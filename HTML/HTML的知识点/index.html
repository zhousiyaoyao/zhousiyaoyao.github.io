HTML的新学知识点

Posted on 2020-05-31 | Post modified: 2020-06-01 | In HTML | 阅读数

行内元素

行内元素设置 width 无效，height 无效（可以设置 line-height）
行内元素设置 margin 和 padding 不会对其他元素产生影响

浏览器渲染原理

  1. 先解析收到的文档，构建一棵dom树，遇到js会先暂停，等js运行完毕
  2. 解析css，生成cssom规则树
  3. 根据dom和cssom树构建渲染树
  4. 渲染树布局和重排
  5. 绘制阶段，遍历渲染树，并调用渲染对象的paint方法把内容显示在屏幕上

async和defer

都可以并行加载，async加载完就执行，执行会阻塞文档解析，defer会等整个文档解析完成后，在DOMContentLoaded事件触发之前完成，按照顺序

文档的预解析

当执行js脚本时，另一个线程会解析剩下的文档，但不会改变dom树，只解析外部资源的引用

css阻塞文档解析

因为js脚本执行可以请求样式信息，如果css没解析完，会得到错误的信息
所以如果cssom还没构建解析好，遇到js脚本，浏览器会延迟js的执行和文档的解析
所以浏览器会先下载和构建cssom，在执行js，再继续文档的解析

FOUC和白屏

FOUC：样式闪烁，css加载时候过长或者css放在文档底部，会先呈现html，无样式内容，然后突然样式呈现
白屏：浏览器需要构建出dom树和cssom树才能进行渲染，所以css加载未完成或者js阻塞文档解析都可以导致白屏

DOMContentLoaded 事件和 Load 事件

DOMContentLoaded：当HTML加载和解析完就触发
Load：所有资源加载完后触发

H5新增功能

  1. 绘画canvas
  2. video和audio元素
  3. localstorage和sessionStorage
  4. webworker
  5. websocket
  6. getCurrentPosition，获得用户当前位置

H5离线存储

基于一个.appcache文件的缓存机制，通过这个文件上解析清单离线存储功能，这些资源就能被存储下来，当离线时，可以展示离线存储的数据，用HTML的头部manifest属性实现，第一次访问app会离线存储，后面每次都会对比文件是否发生变化，变化的化会更新文件

cookie

  1. cookie的domain和path构成了其url，哪些地方可以获取到cookie
  2. 删除cookie需要把cookie的过期时间设为过去的时间
  3. cookie的secure属性设置当请求是HTTPS或者安全协议时，才会被发送。在http协议中无法设置
  4. cookie的httponly属性规定cookie是否能够通过js访问，默认可以，只能通过服务器设置

IndexedDB

  1. 键值对存储，NoSql，支持事务，同源，存储空间大
  2. 异步，localstorage是同步

Label

用于表单的聚焦，当用户选择改标签，会自动转移焦点

实现标签页的通信

  1. Websocket，服务器做中介
  2. ShareWorker，所有页面共享，而webworker只能每个页面一个，属于render进程
  3. localstorage，监听localstorage的变化事件来传递数据
  4. postmessage，跨页面请求数据，安全的跨源通信，需要获的对应标签页的引用

页面可见性api

通过监听网页的可见性，判断网页的卸载，节省资源和耗电，可以停止播放音乐和video，结束轮询和动画
document.visibilityState，hidden页面不可见，visible一部分可见，prerender即将或正在渲染
visibilitychange事件，visibilityState改变触发

Canvas和SVG

Canvas通过js来绘制2D图像，逐像素渲染，缩放时会出现锯齿或者失真
SVG通过XML来绘制2D图像，缩放不会失帧

Chrome瀑布图

紫线表示html解析完，绿线表示开始渲染
越矮越窄，绿线越左，性能越好

serviceWorker 和 PWA

PWA：渐进式网页应用，三大特性service worker离线缓存（存储到cache storage），manifest应用清单（网页出现在手机app清单）和推送通知（服务器push给sw，sw推送给用户）
service worker：浏览器独立于网页运行的脚本，作用于浏览器和服务器之间，相当于代理服务器，能够直接截取和返回用户请求
